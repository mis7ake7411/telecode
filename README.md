**Your AI Terminal Inside Telegram** вҖ” Bridge Telegram with Claude Code and Codex. Execute commands, analyze images, transcribe voice, and chat with AIвҖ”all from your favorite messenger.

- рҹӨ– **Dual AI Engines** вҖ” Switch between Claude Code and Codex on the fly

- рҹҺӨ **Voice Transcription** вҖ” Whisper-powered voice note transcription

- вҡЎ **Real-time Webhooks** вҖ” Instant message processing via Telegram webhooks

## рҹҡҖ Quick Start

### Install

```bash
pip install telecode
```

### Run

```bash
telecode
```

On first run, you'll be prompted for:
- **Telegram Bot Token** (get it from [@BotFather](https://t.me/BotFather))
- **ngrok Auth Token** (optional, for auto-tunneling)

That's it! Find your bot in Telegram and start chatting.

## рҹ“Ў MCP Server (NEW)

Telecode can expose AI tools via the Model Context Protocol, allowing remote agents to access your local AI engines:

### Enable MCP Server

```bash
telecode --enable-mcp
```

The server will display your MCP connection URL:
```
+----------------------------------------------------+
| MCP Server Configuration:                          |
|                                                    |
| URL: https://your-tunnel.ngrok-free.app/mcp/      |
| (Public ngrok URL - share with remote MCP clients)|
|                                                    |
| Available Tools:                                   |
|   - local_claude_code: Execute Claude Code CLI     |
|   - local_codex: Execute Codex CLI                 |
|   - local_cli: Execute shell commands              |
+----------------------------------------------------+
```

### MCP Tools

- **`local_claude_code`** вҖ” Execute prompts with Claude Code CLI
- **`local_codex`** вҖ” Execute prompts with Codex CLI
- **`local_cli`** вҖ” Run shell commands on your server

MCP clients can connect to your server and use these tools remotely. Sessions are independent from Telegram, allowing simultaneous use.

## рҹ’¬ Telegram Commands

| Command | Description |
|---------|-------------|
| `/engine` | Show current AI engine |
| `/claude` | Switch to Claude Code |
| `/codex` | Switch to Codex |
| `/cli <cmd>` | Execute shell command (**disabled by default**, enable with `--enable-cli`) |
| `/tts_on` | Enable TTS audio responses |
| `/tts_off` | Disable TTS responses |

## вҡҷпёҸ Configuration

Configuration is stored in `.telecode` (local) or `~/.telecode` (global):

```bash
TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
TELECODE_ENGINE=claude
TELECODE_ALLOWED_USERS=123456789,@username
TELECODE_ENABLE_MCP=1
TELECODE_ENABLE_CLI=0
TELECODE_MCP_AUTH_TOKEN=change-me
TELEGRAM_WEBHOOK_SECRET_TOKEN=change-me
TELECODE_VERBOSE=1
```

### Key Configuration Options

| Variable | Description | Default |
|----------|-------------|---------|
| `TELEGRAM_BOT_TOKEN` | Bot token from @BotFather | *Required* |
| `TELEGRAM_TUNNEL_URL` | Public webhook URL | Auto via ngrok |
| `TELECODE_ENGINE` | Default engine: `claude` or `codex` | `claude` |
| `TELECODE_ENABLE_MCP` | Enable MCP server | `0` |
| `TELECODE_ALLOWED_USERS` | User whitelist (IDs/@usernames) | *Required* |
| `TELECODE_ENABLE_CLI` | Enable `/cli` and MCP `local_cli` command execution | `0` |
| `TELECODE_MCP_AUTH_TOKEN` | Bearer token required for `/mcp` routes (recommended when MCP enabled) | *(empty)* |
| `TELEGRAM_WEBHOOK_SECRET_TOKEN` | Header secret verified from `X-Telegram-Bot-Api-Secret-Token` | Auto-generated by CLI |
| `TELECODE_VERBOSE` | Enable verbose logging | `0` |
| `TELECODE_TTS` | Enable TTS responses | `0` |

## рҹ”җ Access Control

Restrict bot access to specific users:

```bash
# By user ID (get from @userinfobot)
TELECODE_ALLOWED_USERS=123456789,987654321

# By username
TELECODE_ALLOWED_USERS=@alice,@bob

# Mixed
TELECODE_ALLOWED_USERS=123456789,@alice
```

Do not leave this empty in production; startup now requires explicit allowlist entries.

## рҹӣЎпёҸ Security Defaults

- `TELECODE_ALLOWED_USERS` is required at startup.
- `/cli` and MCP `local_cli` are disabled by default. Enable explicitly with `--enable-cli` (or `TELECODE_ENABLE_CLI=1`).
- Webhook requests are verified by both URL secret and Telegram's header secret token.
- When using MCP remotely, set `TELECODE_MCP_AUTH_TOKEN` and send `Authorization: Bearer <token>`.

## рҹ”§ Troubleshooting

### Bot doesn't respond
- Check webhook URL is accessible
- Run with `-v` for verbose logs
- Verify bot token with [@userinfobot](https://t.me/userinfobot)

### Voice messages fail
```bash
pip install openai-whisper
brew install ffmpeg  # macOS
```

### Invalid bot token error
The CLI will detect invalid tokens and prompt for a new one automatically.

### ngrok authentication failed
Get your auth token from [ngrok.com](https://dashboard.ngrok.com/get-started/your-authtoken) and paste when prompted.
